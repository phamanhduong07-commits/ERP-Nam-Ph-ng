Sprint 4

Variance: BOM kế hoạch vs thực tế

Quan trọng
🔧Sửa G4 — _compute_allocation: thêm chi_phi_ke_hoach từ BOM confirmed vào allocation_by_lsx. Tính variance = actual − standard
🔧GET /production-sessions/{id}/variance — trả về bảng variance per LSX: giấy (kế hoạch vs thực), NVL (kế hoạch vs thực), tổng
🔧Phân tích nguyên nhân — Variance giấy = lượng chênh (kg TT vs kế hoạch) + giá chênh (don_gia_binh_quan vs don_gia_lock)
🔧Frontend: Tab "Variance" trong session detail — bảng màu (xanh/đỏ) so sánh
